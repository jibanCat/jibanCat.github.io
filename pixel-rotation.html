Pixel Rotation in $\texttt{healpy}$¶

healpy is a python package for analyzing Healpix format data on a sphere. Healpix was designed for CMB (cosmic microwave background) data, e.g., (PLANCK)[http://pla.esac.esa.int/pla/] satellite used healpix as the standard format for CMB maps. However, it is still eligible to apply Healpix for other types of images.

When I first time used healpy, I was surprise, "What? How can I rotate the image?". I found out I was so wrong latterly. There is a class object healpy.rotator.Rotator which is able to rotate the map. Healpy's document did not say directly how to use Rotator on a map, but actually we can get some clues in SphericalProj class:

Here is a little tweak to convert cartesian projection to Healpix format, some pixels will loss in this conversion. We should at least pick a Nside to make the number of Healpix pixels smaller than the number of rectangular map. Pixel number of Healpix map is calculated by $12 * \text{nside}^2$.

In [16]:
import healpy as hp
def cart_healpix(cartview, nside):
    '''read in a matrix and return a healpix pixelization map'''
    # Generate a blank Healpix map and angular to pixels
    healpix = np.zeros(hp.nside2npix(nside), dtype=np.double)
    hptheta = np.linspace(0, np.pi, num=cartview.shape[0])[:, None]
    hpphi = np.linspace(-np.pi, np.pi, num=cartview.shape[1])
    pix = hp.ang2pix(nside, hptheta, hpphi)
    
    # re-pixelize
    healpix[pix] = cartview
    return healpix

nside = 256
earth_healpix = cart_healpix(earth, nside)

Now we have a 1-D array in Healpix format which represents a sphere image data. For more detailed description you can read the blog of Dr. Zonca.

There are 3 main projection methods in healpy:

  • hp.mollview: mostly used for visulization for CMB.
  • hp.cartview: return a rectangular matrix.
  • hp.orthview: return two sphere in 3D cartesian coordinate.
  • hp.gnomview: in Gnomonic projection, I seldom use it.

Rotation in Healpix¶

Here is the function for pixel rotation.

In [36]:
def healpix_rotate(healpix_map, rot):
    '''rotate a healpix map: rot=(phi,theta,psi)'''
    ipix = np.arange(len(healpix_map))
    nside = hp.npix2nside(len(healpix_map))    
    # pixel positions to vectors in (x,y,z)
    vec = hp.pix2vec(nside, ipix)
    # rotate a vector with Euler Angles
    rot_vec = (hp.rotator.Rotator(rot=rot)).I(vec)
    # convert vectors back to pixel positions
    irotpix = hp.vec2pix(nside, rot_vec[0], rot_vec[1], rot_vec[2])
    return np.copy(healpix_map[irotpix])

# Let's try out rotation in theta direction
rot = (0,-90,0)

Conclusion¶

One thing should be noted is that there is actually already an rot in visufunc. So you can use cartview with rot=(phi,theta) and return_projected_map=1 to return a rotated cartesian projection map. But for others who want a rotated map in Healpix format, we still have to do the stuff in healpix_rotate function.
